تحرير ملف config.yml

config.yml 
# Config for Netlify CMS
# This file controls the content management system for the blog

backend:
  name: github
  repo: your-username/ai-news-blog
  branch: main
  auth_endpoint: api/auth

# Media files will be stored in the repo under static/images/uploads
media_folder: "static/images/uploads"
public_folder: "/images/uploads"

# Editorial workflow: allows drafts and review
publish_mode: editorial_workflow

# Slug settings
slug:
  encoding: "ascii"
  clean_accents: true
  sanitize_replacement: "-"

# Collections - define content types
collections:
  - name: "blog" # Used in routes, e.g., /admin/collections/blog
    label: "مقالات" # Display name in the CMS UI
    folder: "content/blog"
    create: true # Allow users to create new documents in this collection
    slug: "{{year}}-{{month}}-{{day}}-{{slug}}" # Filename template
    editor:
      preview: true
    fields: # Fields for each document
      - {label: "العنوان", name: "title", widget: "string"}
      - {label: "تاريخ النشر", name: "date", widget: "datetime"}
      - {label: "الصورة المصغرة", name: "thumbnail", widget: "image"}
      - {label: "تصنيف", name: "category", widget: "select", options: ["التعلم العميق", "معالجة اللغات", "رؤية الحاسوب", "الروبوتات", "أخلاقيات الذكاء الاصطناعي", "تطبيقات عملية"]}
      - {label: "الوسوم", name: "tags", widget: "list"}
      - {label: "وصف مختصر", name: "description", widget: "text"}
      - {label: "مميز", name: "featured", widget: "boolean", default: false}
      - {label: "المحتوى", name: "body", widget: "markdown"}

      preview: true

      - {label: "العنوان", name: "title", widget: "string"}

نصائح لتكوين CMS

  • تأكد من تعديل repo: your-username/ai-news-blog إلى مستودع GitHub الخاص بك.
  • يمكنك تخصيص الحقول (fields) لكل نوع محتوى حسب احتياجات مدونتك.
  • الحفاظ على هيكل المجلدات المحددة في التكوين لتجنب مشاكل النشر.
  • تأكد من تمكين موفر المصادقة المناسب في إعدادات Netlify CMS.
  • راجع توثيق Netlify CMS للحصول على مزيد من الخيارات المتقدمة.

تعليمات نشر الموقع على Netlify و GitHub

  1. أنشئ مستودع GitHub جديد

    قم بإنشاء مستودع جديد على GitHub يحمل اسم ai-news-blog أو اسمًا مشابهًا.

  2. قم برفع ملفات المشروع

    استخدم الأوامر التالية لرفع المشروع:

    git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/ai-news-blog.git
git push -u origin main
  3. أنشئ حساب على Netlify

    إذا لم يكن لديك حساب على Netlify، قم بإنشاء حساب جديد على netlify.com.

  4. أنشئ موقع جديد

    في لوحة تحكم Netlify، انقر على "New site from Git" واختر GitHub كمصدر.

  5. اختر المستودع

    اختر مستودع ai-news-blog الذي أنشأته.

  6. إعداد خيارات النشر

    استخدم الإعدادات التالية:

    • Branch to deploy: main
    • Build command: npm run build
    • Publish directory: public
  7. تكوين Netlify CMS

    في لوحة تحكم Netlify، انتقل إلى "Settings" > "Identity" وقم بتمكين خدمة Identity.

  8. تمكين موفر GitHub

    في قسم "Services" ضمن "Identity"، قم بتمكين خدمة "Git Gateway" لتتمكن من الاتصال بـ GitHub.

  9. الوصول إلى لوحة التحكم

    بعد اكتمال النشر، يمكنك الوصول إلى لوحة تحكم CMS عبر https://your-site-name.netlify.app/admin.
